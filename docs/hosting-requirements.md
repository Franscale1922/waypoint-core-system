# Hosting Context — Content Refresh System Requirements

> **For the "Evaluate Hosting Solutions" conversation.**
> This document was generated by the content refresh automation build (March 2026) and documents a hard architectural requirement that must factor into the hosting decision.

---

## The Critical Hosting Requirement

**Auto-deploy on push to `main` is not optional — it is architecturally required.**

The site uses an automated content refresh system (Inngest + OpenAI GPT-4o) that rewrites article files and commits them back to the GitHub repo on a schedule. If the host does not auto-rebuild when `main` receives a push, the refreshed articles will exist in GitHub but never appear on the live site.

This is not a nice-to-have. It is a hard dependency of the content automation system.

---

## How the Content Refresh System Works

```
Inngest cron fires (1st of every month)
  ↓
GPT-4o rewrites stale articles (following content standards)
  ↓
Updated .md files committed to GitHub via API (single atomic commit)
  ↓
Host detects push to main → rebuild triggered automatically
  ↓
Live site updated (~2 min after commit)
```

If step 4 doesn't happen automatically, the loop is broken.

## What to Verify on Any Hosting Platform

| Requirement | Vercel | Netlify | Railway | Notes |
|---|---|---|---|---|
| Auto-deploy on push to `main` | ✅ Default on | ✅ Default on | ✅ Default on | Must be confirmed, not assumed |
| Build command support | ✅ | ✅ | ✅ | `prisma generate && prisma db push && next build` |
| Env var management | ✅ | ✅ | ✅ | Several required (see below) |
| Inngest webhook endpoint | ✅ | ✅ | ✅ | `/api/inngest` must be reachable |
| Persistent DB | ❌ needs external | ❌ needs external | ✅ with volume | Dev uses SQLite (`file:./dev.db`) — prod needs real DB |

**On Vercel:** Settings → Git → Production Branch = `main`, auto-deploy = enabled.
**On Netlify:** Site Settings → Build & Deploy → Auto publishing = on.

## Database Note

Development uses SQLite (`file:./dev.db`). This does **not** work on serverless hosts (Vercel, Netlify) because the filesystem is ephemeral. Production needs a real hosted database:

- **Recommended:** [Neon](https://neon.tech) (serverless Postgres, generous free tier, works perfectly with Prisma)
- **Alternative:** PlanetScale (MySQL-compatible), Supabase (Postgres)
- Update `DATABASE_URL` to the production connection string and change Prisma provider from `sqlite` to `postgresql` in `prisma/schema.prisma`

This is the only schema change needed before production.

## Inngest Production Setup

1. Create a production Inngest app at [app.inngest.com](https://app.inngest.com)
2. Add `INNGEST_SIGNING_KEY` and `INNGEST_EVENT_KEY` to host environment variables
3. Point Inngest webhook to: `https://yourdomain.com/api/inngest`
4. Confirm `content-refresh` cron function shows as active in the Inngest dashboard
